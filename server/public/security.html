Security overview

LynkWell AI API — technical controls and honest compliance language.

What we implement in software

  • Transport: Deploy behind HTTPS (e.g. Vercel) so traffic is encrypted in transit; optional HSTS in production.
  • Secrets: Gemini and LinkedIn credentials live in server environment variables, not in extension source.
  • Auth: Session tokens are signed JWTs (HS256) with explicit algorithm checks; protected routes require a valid Bearer token.
  • Activation: Invite codes and extension secrets compared with timing-safe checks where applicable; brute-force mitigation via rate limits on activation and handoff exchange.
  • HTTP headers: Baseline headers such as X-Content-Type-Options: nosniff, X-Frame-Options: DENY, Referrer-Policy, Permissions-Policy, Content-Security-Policy on served HTML, and strict framing rules.
  • Input limits: JSON body size cap and prompt length limits on AI routes.
  • OAuth: LinkedIn extension-flow uses signed state and short-lived handoff payloads validated server-side.

SOC 2, ISO 27001, and similar frameworks

SOC 2 Type II and ISO 27001 certifications mean an independent auditor has reviewed your organization’s policies, controls, and evidence over a defined period (SOC 2) or your information security management system (ISO 27001). Shipping this API does not, by itself, grant those certifications.

How this maps to “readiness” language Many teams describe an app as implementing controls that commonly appear in SOC 2 / ISO readiness work—encryption in transit, access control, secret handling, logging discipline—then pursue formal audit with their compliance vendor. Use accurate wording in customer contracts: e.g. “designed with security controls aligned to industry practice” unless you hold an active attestation for this service.

Your responsibilities when you go live

  • Rotate JWT_SECRET and activation material if exposed; use strong random values in production.
  • Restrict LinkedIn redirect URLs to the exact production callback(s).
  • Review Vercel (or host) access, audit logs, and who can change environment variables.
  • For enterprise customers, complete your own SOC 2 / ISO program with a licensed assessor if they require attestation.
